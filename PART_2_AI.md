# Part 2: AI Integration

For AI integration there should be static config and dynamic config (contextual files). 

Static config could be in markdown files describing rules, general application context, role context, output strategy and security patterns.

Dynamic config could be generated by application to gather current state and values for given use case.  Internal method or API endpoint to produce dynamic markdown. 

Examples:
1. System: {inject desc from config files}  
   Role: {inject role context}  
   Context: {inject dynamic context based on onboarding state}  
   Constraints: {inject rules, security patterns}  
   
   Lead time is used to calculate the optimal date for restocking a product. 
   Predict average lead time for my business. 
   
   Output: {inject output strategy}
